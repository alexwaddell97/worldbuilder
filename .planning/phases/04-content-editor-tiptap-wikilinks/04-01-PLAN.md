    2. `updateWikilinkLabels(worldId: string, entityId: string, newLabel: string): Promise<void>` — implements the JavaScript fan-out: (a) SELECT all entities in worldId where `content IS NOT NULL` (use `isNotNull(entities.content)`), selecting `{id, content}`; (b) for each row, recursively walk the Tiptap JSON document tree — the tree structure is `{type, content: [...children], attrs: {...}}` — looking for nodes where `node.type === "wikilink"` and `node.attrs?.id === entityId`; (c) when found, set `node.attrs.label = newLabel`; (d) collect the modified rows; (e) for each modified row, run `db.update(entities).set({ content: row.content }).where(and(eq(entities.id, row.id), eq(entities.worldId, worldId)))`. Extract the tree-walk into a local `function walkAndUpdate(node: unknown, entityId: string, newLabel: string): boolean` that mutates in-place and returns true if any change was made. The `content` column is `jsonb` typed as `unknown` — cast to `Record<string, unknown>` at the walk entry point.

    3. `markWikilinksDead(worldId: string, entityId: string): Promise<void>` — same JavaScript fan-out pattern as `updateWikilinkLabels`, but instead of updating `label`, sets `node.attrs.dead = true` for every wikilink node where `node.attrs.id === entityId`. Extract the tree-walk into a local `function walkAndMarkDead(node: unknown, entityId: string): boolean`. SELECT same shape: all entities in worldId where `content IS NOT NULL`. For each modified row, run `db.update(entities).set({ content: row.content }).where(and(eq(entities.id, row.id), eq(entities.worldId, worldId)))`. This function must run BEFORE the entity is deleted (call order enforced in Task 2).
