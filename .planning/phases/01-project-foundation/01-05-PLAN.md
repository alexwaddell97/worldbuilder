### Task 2: Vercel Deployment Configuration

Create `vercel.json` in the project root. Configure production-only deployment per D-11 and D-12:

```json
{
  "git": {
    "deploymentEnabled": {
      "main": true
    }
  }
}
```

This disables automatic Vercel deployments for all branches EXCEPT `main`. Note: Vercel's "Preview Deployments" toggle must also be disabled in the Vercel project settings UI (Dashboard → Project Settings → Git → uncheck "Preview Deployments") — the `vercel.json` setting is code-level enforcement but the UI toggle is the definitive control.

Verify the production build compiles cleanly. Run `npm run build` locally (requires all environment variables set in `.env.local`). Fix any TypeScript errors, missing module errors, or build-time failures before deploying.

Common build-time issues to check:
- Server-only modules (e.g., `src/lib/auth/index.ts`, `src/lib/db/index.ts`) must never be imported in `'use client'` files — Next.js will throw a build error if they are
- `async` Server Components must `await` all async calls — missing `await` causes silent rendering issues
- All `href` and `Link` targets must be strings — no undefined values

Manual deployment steps (execute in order after `npm run build` succeeds):
1. If Vercel CLI is not installed: `npm install -g vercel`
2. `vercel link` — links the local project to a Vercel project (creates one if none exists)
3. Set production environment variables in Vercel: for each variable in `.env.local.example`, run `vercel env add <VAR_NAME> production` and paste the value. Required: `DATABASE_URL`, `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`, `NEXT_PUBLIC_APP_URL`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`
4. `BETTER_AUTH_URL` and `NEXT_PUBLIC_APP_URL` in production should be the Vercel-assigned URL (e.g., `https://worldbuilder-<hash>.vercel.app`) — update after first deployment when the URL is known
5. Push `main` branch to GitHub — Vercel auto-deploys on push. OR run `vercel --prod` for a manual deploy.
6. After deployment: verify the Vercel URL loads, the landing page renders, and the auth flow works (signup → email verify → login → dashboard)

Google OAuth requires the Vercel production URL to be added as an authorized redirect URI in the Google Cloud Console: `https://<vercel-url>/api/auth/callback/google`. Add this after the production URL is known.
