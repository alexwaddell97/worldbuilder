Do NOT use `revalidateTag` (single-arg form is a TypeScript error in Next.js 16); `updateTag` is correct for read-your-writes and `revalidatePath` drops the client Router Cache so the dashboard re-renders after navigation.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Server Action | FormData and worldId arguments are attacker-controlled until validated |
| Server Action → DB | All mutations cross into persistent state; must be owner-scoped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | updateWorldAction / deleteWorldAction / togglePrivacyAction worldId arg | mitigate | Every mutation where clause includes `and(eq(worlds.id, worldId), eq(worlds.ownerId, session.user.id))` — IDOR prevented; a non-owner's worldId matches zero rows |
| T-02-02 | Spoofing | session.user.id source | mitigate | ownerId always read from `auth.api.getSession`; client-passed ownerId never trusted |
| T-02-03 | Tampering | FormData mass assignment | mitigate | Zod schema allowlist accepts only `name` and `description`; `isPublic`/`slug`/`ownerId` set server-side only |
| T-02-04 | Tampering | slug injection via name | mitigate | `slugify(name, { strict: true })` strips unsafe chars; composite unique constraint enforces per-user uniqueness at DB layer |
| T-02-05 | Information Disclosure | unauthenticated action invocation | mitigate | Session verified at top of every action; throws "Unauthorized" when null |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan — all packages (zod, slugify) already vetted [OK] in RESEARCH Package Legitimacy Audit |
</threat_model>
