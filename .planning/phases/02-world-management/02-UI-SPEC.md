## Spacing Scale

Declared values (all multiples of 4). Uses Tailwind CSS v4 with shadcn CSS variable tokens.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element gaps |
| sm | 8px | Compact element spacing, nav item padding (py-2 px-2) |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Card internal padding, section spacing |
| xl | 32px | Page content padding (p-8 = 32px — matches existing dashboard) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions:
- Sidebar widths: 240px expanded / 56px collapsed (pre-established in Phase 1 — do not change)
- Sidebar header height: 56px (h-14) — pre-established, do not change
- Touch targets: minimum 44px for interactive controls (icon-only buttons, toggle switch)
- World card action menu trigger: 32px hit area (icon-only, Tailwind `h-8 w-8`)

Source: `src/components/layout/sidebar.tsx` — existing dimensions are locked.

## Typography

All sizes use Geist Sans. Weights are exactly two: regular (400) and semibold (600).

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Display | 24px | 600 | 1.2 | `text-2xl font-semibold tracking-tight` |
| Heading | 18px | 600 | 1.3 | `text-lg font-semibold` |
| Body | 16px | 400 | 1.5 | `text-base` |
| Label / Secondary | 14px | 400 | 1.4 | `text-sm text-muted-foreground` |

Source: Existing `DashboardPage` uses `text-2xl font-semibold tracking-tight` for the page heading and `text-sm text-muted-foreground` for secondary text — carry these forward exactly.

Rules:
- Never introduce a third weight (no medium/700/800)
- Never introduce a fifth size tier
- `tracking-tight` applies to Display and Heading only
- Mono font (Geist Mono) is reserved for slug display only: world slug shown in card and edit form uses `font-mono text-sm`

## Color

All colors are CSS variables. No hardcoded hex/rgb/hsl values anywhere except in `globals.css`.

| Role | CSS Variable | HSL Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | `--background` | `0 0% 99%` | Page backgrounds, sidebar background |
| Secondary (30%) | `--card` / `--muted` | `0 0% 100%` / `0 0% 96%` | World cards (`--card`), nav active states and input backgrounds (`--muted`) |
| Accent (10%) | `--primary` | `0 0% 9%` | Primary CTA button fill only |
| Destructive | `--destructive` | `0 84% 60%` | Delete button in AlertDialog only |

Accent (`--primary`, near-black) is reserved for:
1. "Create World" primary button (filled)
2. "Save Changes" button in edit dialog (filled)

All other interactive elements (secondary buttons, icon buttons, nav links, toggle) use `--muted` or `--secondary` surfaces with `--foreground` text. No accent on hover states, badges, or decorative elements.

Privacy badge colors:
- Private: `bg-muted text-muted-foreground` (neutral, no accent)
- Public: `bg-muted text-muted-foreground` with `Globe` icon (same neutral treatment — privacy is not a call-to-action)

Source: `src/app/globals.css` — all token values locked from Phase 1 (Decision D-14, D-15, D-17).

## Surface Specifications

**Focal point:** "Create World" primary button in the dashboard header row. This is the single highest-priority interactive element on the dashboard — all surrounding layout defers to it visually.

### 1. World Dashboard (`/dashboard`)

**Layout:** Full-width content area (`flex-1 overflow-y-auto`), padding `p-8` (matches existing dashboard shell).

**Structure (top to bottom):**
1. Page header row: `text-2xl font-semibold tracking-tight` heading "Your Worlds" + "Create World" primary Button aligned right (`flex items-center justify-between`)
2. World card grid: `grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3` — responsive, 16px gap
3. Empty state: centered, when zero worlds exist

**Heading row:** `mb-6` below it before grid.

### 2. World Card (`WorldCard`)

**Container:** shadcn Card with `p-6` internal padding, `hover:shadow-sm transition-shadow` on the card root.

**Structure:**
```
[Card]
  [Card Header — flex row, items-start, justify-between]
    [Left: name + slug stack]
      World name — text-base font-semibold
      Slug — font-mono text-sm text-muted-foreground (e.g. /worlds/my-world)
    [Right: DropdownMenu trigger — icon-only Button variant="ghost" size="icon"]
      MoreHorizontal icon (16px)
  [Card Content]
    Description — text-sm text-muted-foreground, line-clamp-2, mb-4
    [Footer row — flex items-center gap-2]
      Privacy Badge (Private or Public)
      [Spacer — flex-1]
      Privacy Toggle (Switch, sm size)
```

**DropdownMenu items:**
- "Edit" — Pencil icon (14px), `text-sm`
- "Delete" — Trash2 icon (14px), `text-sm text-destructive` — opens AlertDialog, does NOT execute delete directly

**Privacy Badge:**
- Variant: `outline` (shadcn Badge)
- Private: Lock icon (12px) + "Private" text
- Public: Globe icon (12px) + "Public" text
- No color change between states — both use `variant="outline"` with `text-muted-foreground`

### 3. Create World Dialog (`CreateWorldDialog`)

**Trigger:** "Create World" Button, `variant="default"` (primary fill), Plus icon (16px) prepended. Top-right of dashboard header row.

**Dialog dimensions:** `max-w-md` width.

**Dialog title:** "Create a new world"

**Dialog description (shadcn DialogDescription):** "Give your world a name. A URL slug is generated automatically."

**Form fields (top to bottom):**

| Field | Component | Attributes |
|-------|-----------|------------|
| World Name | Input | `name="name"`, `placeholder="e.g. The Shattered Realm"`, `autoFocus`, max 100 chars |
| Description | Textarea | `name="description"`, `placeholder="What is this world about? (optional)"`, `rows={3}`, max 500 chars |
| Slug preview | Read-only display (not an Input) | `font-mono text-sm text-muted-foreground`, updates live as name is typed — shows `/worlds/{slug}` format. Visually: `bg-muted rounded-md px-3 py-2 text-sm font-mono text-muted-foreground` |

**Field error display:** `text-sm text-destructive mt-1` below each field. Powered by `useActionState` error shape.

**Footer buttons (DialogFooter):**
- Discard changes: `variant="outline"` — closes dialog, no mutation
- Create World: `variant="default"` (primary) — `disabled={pending}`, text changes to "Creating..." when pending

### 4. Edit World Dialog (`EditWorldDialog`)

Same structure as Create World Dialog with these differences:

**Dialog title:** "Edit world"

**Dialog description:** "Update your world's name or description. The slug cannot be changed after creation."

**Form fields:**
- World Name: pre-filled with current name
- Description: pre-filled with current description
- Slug: read-only display, same format as Create — slug is immutable after creation

**Footer buttons:**
- Discard changes: `variant="outline"`
- Save Changes: `variant="default"`, `disabled={pending}`, text changes to "Saving..." when pending

### 5. Delete World Confirmation (`DeleteWorldDialog`)

Uses shadcn AlertDialog (not Dialog). AlertDialog is the correct component for destructive confirmations — it has a distinct visual treatment and blocks page interaction.

**Trigger:** "Delete" item in WorldCard DropdownMenu.

**AlertDialog title:** "Delete world?"

**AlertDialog description:** "This will permanently delete **{worldName}** and all its content. This action cannot be undone."
- World name rendered in `<strong>` within the description
- Max description width: `max-w-sm`

**Footer buttons:**
- Keep world: `variant="outline"` — always left button
- Delete: `variant="destructive"`, label "Delete world", `disabled={pending}`, text changes to "Deleting..." when pending

**No secondary confirmation required** (e.g., no "type world name to confirm") — a single AlertDialog click is sufficient for v1.

### 6. Privacy Toggle (`PrivacyToggle`)

**Component:** shadcn Switch + Label.

**Placement:** Bottom-right of WorldCard footer row.

**Interaction:** Fires `togglePrivacyAction` Server Action on change. Uses `useOptimistic` for instant visual feedback before server responds.

**Label:** Rendered visually via the Privacy Badge to the left — the Switch itself has no visible text label, but has `aria-label="Toggle world privacy"` for accessibility.

**Switch state:** `checked={isPublic}` — checked = public, unchecked = private.

**Pending state:** Switch is `disabled` while the optimistic update is in flight.

### 7. World Detail Page Shell (`/worlds/[slug]`)

Phase 2 delivers only the shell. Entities come in Phase 3.

**Layout:** Same `flex-1 overflow-y-auto` main area, `p-8` padding.

**Structure (top to bottom):**
1. Breadcrumb: `text-sm text-muted-foreground` — "Your Worlds / {World Name}" — "Your Worlds" is a Link back to `/dashboard`
2. Page header row: World name `text-2xl font-semibold tracking-tight` + Edit button (`variant="outline" size="sm"`) + Delete button (`variant="outline" size="sm" text-destructive`) — right-aligned, `gap-2`
3. World description: `text-sm text-muted-foreground` max-w-prose, `mt-2 mb-6`
4. Metadata row: Privacy badge + slug display (`font-mono text-sm text-muted-foreground`) — `flex items-center gap-3`, `mb-8`
5. Separator
6. Placeholder section: `mt-8`, centered, `text-muted-foreground text-sm` — "Entities and content coming in Phase 3."

**Sidebar behavior in world context:** The sidebar should show the world name as a contextual header when inside `/worlds/[slug]/...`. Add a new nav section below the world name reading "Entities (Phase 3)" in `text-xs text-muted-foreground` — this is a placeholder that communicates the structure without requiring real nav items.

## Accessibility Contract

- All Dialogs and AlertDialogs use shadcn/Radix primitives — focus trap, escape key dismiss, and ARIA attributes are handled automatically.
- Privacy Switch: `aria-label="Toggle world privacy"` — no visible text label on the switch itself.
- DropdownMenu trigger (WorldCard): `aria-label="World options for {worldName}"`.
- Delete confirmation: AlertDialog (not Dialog) — semantically communicates destructive intent to screen readers via `role="alertdialog"`.
- World card slug: wrapped in `<span aria-label="World URL: /worlds/{slug}">` for screen reader clarity.
- Form errors: associated with fields via `aria-describedby` pointing to the error element `id`.
- Keyboard: All interactive elements reachable via Tab. Dialog close on Escape. DropdownMenu navigable with arrow keys (Radix handles this).

## Animation Contract

Consistent with Apple-clean aesthetic: subtle, purposeful, never decorative.

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Sidebar width | `transition-[width] duration-200 ease-in-out` | 200ms | ease-in-out (matches Phase 1) |
| Dialog open/close | shadcn default fade + scale | 150ms | ease-out (Radix default) |
| WorldCard hover shadow | `transition-shadow` | 150ms | ease-out |
| Privacy Switch toggle | shadcn default slide | 100ms | ease (Radix default) |
| DropdownMenu open | shadcn default fade | 100ms | ease-out (Radix default) |

No custom keyframes. No entrance animations on page load. No loading spinners — use disabled button with text change ("Creating...") for pending states.
