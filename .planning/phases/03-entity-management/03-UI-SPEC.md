## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` | Icon gaps, inline badge gaps, tight icon grid spacing |
| sm | 8px | `gap-2`, `px-2 py-2` | Nav item padding, compact button gaps, badge spacing |
| md | 16px | `gap-4`, `space-y-4` | Default form field spacing, card inner padding |
| lg | 24px | `gap-6`, `p-6` | Card padding, section spacing |
| xl | 32px | `gap-8` | Layout gaps between page sections |
| 2xl | 48px | `mb-12` | Major section breaks (page header → content) |
| 3xl | 64px | `py-16` | Page-level top padding |

**Exceptions:**
- Icon picker grid cells: `p-1.5` (6px) — intentionally sub-8px for dense 7-column grid. Only exception in this phase.
- Form field label-to-input gap: `space-y-1.5` (6px) — inherits Phase 2 `WorldForm` pattern. Matches shadcn form convention.

## Typography

Source: Geist Sans loaded via `--font-sans`. All sizes via Tailwind text utilities. No custom `font-size` tokens.

| Role | Size | Tailwind Class | Weight | Line Height |
|------|------|----------------|--------|-------------|
| Page heading | 24px | `text-2xl` | 600 — `font-semibold` | 1.2 |
| Card / dialog title | 16px | `text-base` | 600 — `font-semibold` | 1.3 |
| Body / description | 14px | `text-sm` | 400 — `font-normal` | 1.5 |
| Caption / label / slug | 12px | `text-xs` | 400 — `font-normal` | 1.4 |

**Rules:**
- Maximum 4 sizes, 2 weights. Do not introduce `text-lg` (18px) or `text-xl` (20px) — not in the established scale.
- Monospace font (`font-mono text-sm text-muted-foreground`) for slug displays — inherits Phase 2 `WorldCard` pattern.
- Dialog titles use `DialogTitle` which renders `text-base font-semibold` via shadcn defaults.

## Color

All values are CSS variable references. Never use hardcoded hex or HSL values. Reference via Tailwind utilities only.

| Role | CSS Variable | Approx. | Usage |
|------|-------------|---------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 99%)` ≈ #FCFCFC | Page background, all layout surfaces |
| Secondary (30%) | `--card` / `--muted` / `--secondary` | `hsl(0 0% 96–100%)` | Cards, sidebar context block, tab hover, `hover:bg-muted` |
| Accent (10%) | `--primary` | `hsl(0 0% 9%)` ≈ #171717 | CTAs, active tab underline, selected icon ring, primary button bg |
| Destructive | `--destructive` | `hsl(0 84% 60%)` ≈ #EF4444 | Delete actions only, required field asterisk |

**Accent (`--primary`) reserved for:**
- `Button` variant="default" background (primary CTAs: "New [Type]", "New entity type", "Create", "Save changes")
- Active entity type tab: `border-b-2 border-foreground text-foreground font-medium`
- Selected icon cell in icon picker: `ring-1 ring-ring bg-muted`
- `Switch` checked state (Radix default)
- Active tag filter badge: `Badge` with ring indicator

**Muted foreground (`--muted-foreground`):**
- Inactive tab text, placeholder text, slug monospace text, tag `×` remove button, empty state body text
- Sidebar entity type icons when inactive

## Interaction Patterns

### Tags Input
1. User types in tags `Input`. On `,` keydown or `Enter` (only inside tags field): split on comma, trim, lowercase each, add as badge, clear input.
2. Duplicate tags silently discarded (case-insensitive comparison).
3. Each badge: `Badge variant="secondary"` + inline `<button type="button">` with `X size={10}` to remove. Remove button: `text-muted-foreground hover:text-foreground transition-colors`.
4. Before form submit: `<input type="hidden" name="tags" value={tags.join(",")} />`.
5. Server Action normalizes: `raw.split(",").map(t => t.trim().toLowerCase()).filter(Boolean)`.

### Search Bar
1. Controlled `Input` pre-populated from URL `searchParams.get("q") ?? ""`.
2. `onChange`: debounce 300ms, then `router.push` with updated `?q=` param (preserve `?tag=` if active).
3. Input clearing (empty value): remove `q` param via `router.push`.

### Tag Filter
1. Single active tag at a time (Phase 3 — no multi-select).
2. `Popover` trigger: "Filter by tag" (inactive) / "Tag: [name] ×" (active). `×` removes `tag` param.
3. Popover content: all distinct tags across entities of this type, rendered as clickable `Badge` buttons. Clicking a tag: close popover, `router.push` with `?tag=[name]` (preserve `?q=` if active).
4. Clicking the already-active tag: deselect (remove `tag` param).

### Entity Type Tabs
1. `<nav>` with `<Link>` — not Radix `Tabs`. URL is the source of truth.
2. Active: `border-b-2 border-foreground text-foreground font-medium` with `-mb-px` to overlap the nav's `border-b`.
3. Inactive: `text-muted-foreground hover:text-foreground`.
4. Tab icon: `LucideIcon size={14}` inline-left of type name with `gap-1.5`.

### Icon Picker Selection
1. Click trigger → opens Popover.
2. Click icon cell → calls `onChange(iconName)`, closes Popover.
3. Selected cell state: `bg-muted ring-1 ring-ring` (visible highlight without additional color).
4. No keyboard trap — Radix `Popover` handles focus management.
